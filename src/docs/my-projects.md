# Mes projets

## Projet 1 — Système de Gestion de Coopérative Agricole
- Type : Application desktop
- Statut : Projet académique
- Technologies :
  - Java
  - Swing / AWT
  - MySQL
  - JDBC
  - UML
  - JFreeChart
- GitHub : https://github.com/salmabenboukhris/Projet_Cooperative.git

### Description
Application de bureau destinée à une coopérative agricole. Elle permet de gérer le cycle de vente des produits agroalimentaires, de suivre les stocks des agriculteurs et d’analyser les performances commerciales via des outils statistiques.

### Fonctionnalités principales
- Gestion CRUD des entités principales
- Filtrage et recherche SQL paramétrée
- Authentification
- Hachage des mots de passe
- Visualisation statistique avec JFreeChart
- Navigation fluide entre les modules

### Architecture / conception
- Modélisation UML
- Architecture DAO
- Séparation UI / logique métier / accès aux données

### Points forts
- Bonne séparation des couches
- Interface de gestion complète
- Sécurité basique
- Analyse statistique

## Projet 2 — Système de suivi de projets
- Type : Application web
- Statut : Projet académique

### Description
Application web permettant de gérer et suivre des projets au sein d’une entreprise de services. Le système centralise toutes les informations liées aux projets (description, client, dates, budget) et facilite la coordination entre les différents intervenants.

### Fonctionnalités principales
- Gestion des projets (création, modification, consultation)
- Gestion des phases d’un projet
- Suivi de l’avancement (état des phases)
- Gestion des utilisateurs avec rôles (directeur, chef de projet, secrétaire, comptable…)
- Gestion des livrables associés aux phases
- Suivi de facturation et paiement des phases
- Recherche et filtrage des projets

### Points forts
- Gestion multi-rôles avec permissions
- Organisation des projets en phases
- Suivi complet : technique + financier
- Centralisation des données et historique des projets

### Remarque
Utiliser seulement les informations confirmées. Ne pas inventer les fonctionnalités absentes.

## Projet 3 — Restaurant Manager
- Type : Application web
- Statut : Projet académique
- Technologies :
  - Spring Boot
  - Thymeleaf
  - Spring Security
  - Chart.js
- GitHub : https://github.com/abajadda2955-web/mini-projet-spring-thymeleaf.git

### Description
Application web complète de gestion des réservations d’un restaurant.

### Fonctionnalités principales
- Création et suivi des réservations
- Attribution automatique des tables
- Gestion des statuts :
  - CONFIRME
  - HONORE
  - ANNULE
  - NO_SHOW
- Filtrage par date, statut, service et zone
- Dashboard avec statistiques
- Authentification et gestion des rôles

### Architecture
- MVC
- Controller / Service / Repository
- DTO
- Gestion globale des erreurs

### Points forts
- Sécurité avec Spring Security
- Dashboard dynamique
- Validation et séparation des couches
- Architecture propre
